## ?? Getting Started

### Step 1: Start Backend
```bash
cd BudgetBackend
dotnet run
# API running on http://localhost:5000
```

### Step 2: Login & Get Token
```bash
curl -X POST http://localhost:5000/api/admin/login \
  -H "Content-Type: application/json" \
  -d '{
    "email": "admin@example.com",
    "password": "password123"
  }'

# Response:
# {
#   "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
#   "expiresIn": "2025-01-15T20:00:00Z",
#   "id": 1,
#   "userType": "Admin"
# }
```

### Step 3: Use Token in Requests
```bash
curl -X GET http://localhost:5000/api/admin \
  -H "Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
```

## ?? Common Issues & Solutions

### ? Token Expired
```
Response: 401 Unauthorized
Solution: User must login again to get new token
```

### ? Access Denied
```
Response: 403 Forbidden
Solution: User role doesn't have permission for that resource
```

### ? Invalid Token
```
Response: 401 Unauthorized with "Invalid token"
Solution: Token may be malformed or tampered
```

### ? CORS Error
```
Solution: Add frontend URL to CORS policy in Program.cs
```

## ? Key Features Summary

? **Admin User**
- Login with JWT token
- Full CRUD access to all resources
- Can manage other admins
- Can delete any resource

? **Societe User**
- Login with JWT token
- Can view/edit only own company data
- Can create/read/update resources (not delete)
- Can update LigneFinanciere amounts
- Can manage own company's UserSociete
- Cannot access other companies' data

? **Security**
- Token validation on every request
- Role-based access control
- Data ownership validation
- Secure email endpoints (forgot/reset password)
- Configurable token expiration
