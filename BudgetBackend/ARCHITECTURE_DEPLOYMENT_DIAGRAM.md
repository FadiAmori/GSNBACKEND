## Data Flow: User Login Example

```
1. FRONTEND (Angular on Vercel)
   User enters: username@example.com, password123
       ?
       ?
   POST https://backend-url/api/admin/login
   Content-Type: application/json
   {
     "username": "username@example.com",
     "password": "password123"
   }

2. CORS POLICY CHECK
   ? Request origin: https://frontend-domain.com
   ? Allowed origin: In Cors__AllowedOrigins
   ? Method: POST (allowed)
   ? Headers: Content-Type (allowed)
   ? CORS Check PASSED

3. BACKEND (ASP.NET Core on Railway/Render/Azure)
   Route: POST /api/admin/login
   Controller: AdminController.Login()
       ?
       ?
   AdminService.AuthenticateAsync()
       ?
       ??? DATABASE QUERY
       ?   SELECT * FROM Admin WHERE Email = 'username@example.com'
       ?   PostgreSQL returns user record
       ?
       ??? PASSWORD VALIDATION
       ?   Compare password hash with provided password
       ?   ? Match
       ?
       ??? JWT TOKEN GENERATION
       ?   Use JwtSettings__SecretKey to sign token
       ?   Token includes:
       ?   - User ID
       ?   - Email
       ?   - Role (Admin, User, etc)
       ?   - Expiration: 8 hours
       ?
       ??? RETURN JSON
           {
             "token": "eyJhbGc...",
             "user": { "id": 1, "email": "..." },
             "expiresIn": 28800
           }

4. FRONTEND RECEIVES TOKEN
   localStorage.setItem('token', response.token)
   Redirect to dashboard

5. SUBSEQUENT REQUESTS
   GET https://backend-url/api/societe
   Authorization: Bearer eyJhbGc...
       ?
       ?
   BACKEND JWT VALIDATION
   - Verify signature using JwtSettings__SecretKey
   - Check token not expired
   - Extract claims (user id, role, etc)
   - ? Valid ? Process request
   - ? Invalid ? Return 401 Unauthorized

6. DATA RETRIEVAL
   AuthController extracts user ID from JWT claims
   SocieteService.GetByUserAsync(userId)
       ?
       ??? DATABASE QUERY
       ?   SELECT * FROM Societe WHERE UserId = ?
       ?   
       ??? RETURN DATA
           [
             { "id": 1, "name": "Company A", ... },
             { "id": 2, "name": "Company B", ... }
           ]

7. FRONTEND RENDERS DATA
   Display companies in Angular dashboard
```

## Configuration Flow

```
Application Startup (Program.cs)

1. GET ENVIRONMENT
   ASPNETCORE_ENVIRONMENT = "Production" (from env var)
   
2. LOAD CONFIGURATION (In Priority Order)
   
   a. appsettings.json (Base)
      {
        "ConnectionStrings": {...},
        "JwtSettings": {...},
        "Cors": {...}
      }
   
   b. appsettings.Production.json (Overrides)
      {
        "ConnectionStrings": {...},
        "Cors": { "AllowedOrigins": ["https://real-domain"] }
      }
   
   c. Environment Variables (Highest Priority)
      ConnectionStrings__DefaultConnection = "real-value"
      JwtSettings__SecretKey = "real-value"
      Cors__AllowedOrigins__0 = "https://real-domain"

3. MERGE & USE
   Final Configuration = Base + Env-Specific + Env Vars
   
   ????????????????????????????????????
   ?   FINAL CONFIGURATION IN RAM     ?
   ????????????????????????????????????
   ? ConnectionStrings: (from env var)?
   ? JwtSettings: (from env var)      ?
   ? Cors: (from env var)             ?
   ? Logging: (from appsettings)      ?
   ????????????????????????????????????

4. USE IN APPLICATION
   var connectionString = config.GetConnectionString("DefaultConnection")
   var secretKey = config["JwtSettings:SecretKey"]
   var origins = config.GetSection("Cors:AllowedOrigins").Get<string[]>()
```
